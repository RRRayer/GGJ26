| 문서 설명: 게임 구현에 필요한 아트/애니메이션/VFX/오디오 리소스 요구사항을 정의한다.

## 개요
본 문서는 핵심 게임 플레이 구현에 필요한 리소스를 실무 제작 기준으로 정리한다.

## 요구사항
### **아트 리소스**
1. 캐릭터 기본 모델: 플레이어/AI 공용 마네킹 1종. 경로 `Assets/03.Prefabs/Character/`.
2. 가면 프리팹: 빨강/초록/파랑 3종. 경로 `Assets/03.Prefabs/Mask/`.
3. 맵 오브젝트: 광장 구조물, 미러볼, 천막. 경로 `Assets/03.Prefabs/World/`.
4. 무기 모델: 술래 스턴건 1종. 경로 `Assets/03.Prefabs/Weapon/`.
5. UI 아이콘: 키보드 키셋, 댄스 액션 아이콘 4종. 경로 `Assets/05.Arts/UI/`.

### **애니메이션 리소스**
1. 기본 동작: Idle/Walk/Run/Jump 세트.
2. 특수 동작: 스턴건 발사, 피격, 술래 페널티 모션.
3. 댄스 동작: 가면 댄스/단체 댄스용 4종.

### **VFX 리소스**
1. 가면 교체 이펙트.
2. 스턴건 발사/피격 이펙트.
3. 단체 댄스 연출 이펙트(조명/플래시).

### **오디오 리소스**
1. BGM: 평상시 BGM, 단체 댄스 BGM. 경로 `Assets/04.Audios/Clips/BGM/`.
2. SFX: 스턴건, 가면 교체, AI 행동, 승패 효과음. 경로 `Assets/04.Audios/Clips/SFX/`.

## 규칙
### **파일/명명 규칙**
1. 파일명 규칙: `Type_Category_Name_Variant`.
2. 예시: `UI_Icon_Dance_01`, `SFX_Weapon_Taser_Shot`.
3. 경로 표기 시 구분자는 `/`를 사용한다.

### **포맷/용량 기준**
1. 텍스처: `PNG`, 권장 1024x1024 이하, 단일 파일 2MB 이하.
2. 오디오: `WAV` 원본, 런타임 변환은 `OGG`, 단일 파일 10MB 이하.
3. 애니메이션: `FBX`, 루트 모션 여부를 파일명 접미사로 표기한다(`_RM`, `_InPlace`).

### **구현 연동 규칙**
1. 단체 댄스 BGM은 재생 위치를 저장하고 다음 이벤트에서 이어서 재생한다.
2. 가면 리소스는 3색 고정값을 우선 적용한다.
3. 리소스 교체 가능성을 위해 프리팹 참조는 데이터 테이블로 관리한다.

## 예외 케이스
1. 외부 에셋 라이선스 제약으로 상용 배포가 불가한 경우 대체 리소스로 교체한다.
2. 성능 이슈 발생 시 텍스처 해상도와 VFX 파티클 수를 단계적으로 축소한다.
3. 납기 지연 시 후순위 연출 리소스(고급 VFX/추가 댄스)는 릴리스 이후로 이관한다.

## 테스트 시나리오
1. 리소스 로드 테스트: 게임 시작 후 필수 프리팹/아이콘/사운드 누락이 없는지 확인한다.
2. 이벤트 연동 테스트: 단체 댄스 시작/종료 시 BGM, VFX, 애니메이션이 동기화되는지 확인한다.
3. 성능 테스트: 저사양 기준 프레임에서 이펙트와 오디오 동시 재생 시 프레임 드랍 여부를 확인한다.
4. 빌드 테스트: Standalone 빌드에서 경로 참조 오류와 포맷 호환성 오류를 확인한다.

## 우선순위(MVP/후순위)
1. MVP: 기본 캐릭터, 가면 3종, 스턴건, 기본 애니메이션, 핵심 SFX/BGM, 기본 UI 아이콘.
2. 후순위: 고급 VFX, 추가 댄스 모션, 확장 UI 아이콘 세트.

## 영향 범위
### **개발 영향도**
1. 데이터 테이블 기반 리소스 참조 시스템이 필요하다.
2. 오디오 재생 위치 저장 로직과 이벤트 동기화 로직이 필요하다.

### **리소스 영향도**
1. 아트: 캐릭터/가면/월드/아이콘 제작 및 통합.
2. 애니메이션: 기본/특수/댄스 모션 제작 일정 확보 필요.
3. 사운드: 모드별 BGM/SFX 분리 관리 및 루프 정책 정의 필요.
