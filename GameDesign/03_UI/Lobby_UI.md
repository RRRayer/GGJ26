| 로비(타이틀) UI 구성과 방 생성/탐색 흐름을 정의한다.

## 개요
로비(타이틀) 화면 UI와 방 생성/탐색 흐름을 정의한다. 상세 요소는 이 문서를 기준으로 확장한다.

## 핵심 목표/승패 조건
1. 해당 문서는 로비 UI 정의가 목적이므로 승패 조건은 `01_Project/Main_Design.md`와 모드 문서를 따른다.

## 핵심 규칙/메카닉
1. 타이틀 화면에는 Host, Public, Exit 3가지 버튼이 존재한다.
2. Host는 방 만들기 플로우로 진입한다.
3. Public은 방 목록 탐색 플로우로 진입한다.
4. Exit는 게임을 종료한다.

## 게임 흐름
1. 앱 실행 후 타이틀 화면 진입.
2. Host 또는 Public 중 하나를 선택.
3. 방 생성/참가 완료 시 게임 시작 대기 또는 매치 진입.

## 입력/조작
1. 버튼 클릭: 마우스.
2. 화면 종료: ESC.

## UI/연출
### 타이틀 화면 UI
1. 버튼 구성: Host, Public, Exit.

### Host: 방 만들기
1. 입력 항목: 방 제목, 인원 수, 모드, 비밀방, 비밀번호.
2. 만들기 버튼 클릭 시 방이 생성된다.
3. 비밀방이 활성화된 경우 비밀번호 입력은 필수다.

### Public: 방 찾기
1. 방 목록 항목: 방 번호, 제목, 참가 버튼.
2. Public 화면에서도 방 만들기 버튼을 제공한다.

## 리소스/기술 메모
1. 방 목록은 최신 순 또는 조건 검색 필터를 지원할지 결정이 필요하다.

## 우선순위(MVP/후순위)
1. MVP: Host/ Public/ Exit 버튼, 방 생성 폼, 방 목록, 참가 버튼.
2. 후순위: 검색/필터, 방 목록 정렬, 썸네일.

## 확정 사항
1. 방 인원 수 범위는 4명으로 고정.
2. 모드 선택은 Classic/Raccoon 2종이며, 확장 가능성 존재.
