| 문서 설명: 메인 화면부터 인게임/결과까지 UI 흐름과 화면 구성을 정의한다.

## 개요
UI 흐름과 화면 구성을 정리한다. 상세 UI 요소는 이 문서를 기준으로 확장한다.

## 핵심 목표/승패 조건
1. 해당 문서는 UI 흐름 정의가 목적이므로 승패 조건은 `01_Project/Main_Design.md`와 모드 문서를 따른다.

## 핵심 규칙/메카닉
1. 메인 메뉴에서 게임 시작, 스킨 변경, 설정, 종료를 제공한다.
2. 인게임 UI는 타이머, 미션 알림, 조작 안내, 술래 전용 HUD를 포함한다.
3. 로비 상세 UI는 `03_UI/Lobby_UI.md`를 따른다.

## 게임 흐름
1. 앱 실행 후 메인 화면 진입.
2. 게임 시작 클릭 시 로딩 UI 및 매치메이킹.
3. 매치메이킹 성공 시 인게임 UI 표시.
4. 게임 종료 시 결과 화면 표시.
5. 메인 화면으로 복귀.

## 입력/조작
1. 스킨 화면 이동: 좌우 화살표 버튼 클릭 또는 A, D, Left Arrow, Right Arrow.
2. 닫기: ESC.

## UI/연출
### 타이틀 화면 UI
1. 타이틀 화면 상세는 `03_UI/Lobby_UI.md`를 따른다.
2. 스킨 선택 화면은 타이틀 화면의 일부이며, 스킨 미리보기/좌우 이동/장착/닫기를 포함한다.
3. 스킨 목록은 Circular Linked List로 순환 이동한다.
4. 스킨 변경 시 캐릭터 몸은 고정이며 가면만 교체된다.

### 인게임 UI
1. 타이머, 미션 알림, 댄스 아이콘, 보이스 상태, 술래 조준선, 스킬 쿨타임을 표시한다.
2. 결과 화면은 인게임 UI 범주에 포함한다. 승패 텍스트, 통계, 메인으로 버튼을 표시한다.

## 리소스/기술 메모
1. 결과 UI 통계 항목은 모드별로 분기한다.

## 우선순위(MVP/후순위)
1. MVP: 메인 화면, 인게임 HUD, 결과 화면.
2. 후순위: 스킨 화면 애니메이션, 연출 강화.

## 질문/가정사항
1. 결과 화면 통계 항목은 고정인가, 모드별로 커스터마이즈하는가.
